## 🔥 معرفی پروژه

**leastBOT** یک ابزار حرفه‌ای برای ساخت و مدیریت Reverse SSH Tunnel بین سرور ایران و خارج است.

این اسکریپت مخصوص سناریوهایی طراحی شده که:

- 🇮🇷 سرور ایران پشت فایروال یا محدودیت شبکه قرار دارد  
- 🌍 نیاز دارید پنل یا پورت سرور ایران از طریق سرور خارج در دسترس عموم باشد  
- 🔁 اتصال پایدار با Reconnect خودکار نیاز دارید  
- 🤖 ربات تلگرام یا سرویس‌های خارج از ایران امکان اتصال مستقیم به سرور ایران را ندارند
- ✅ یکی از مشکلات دوستانی که از ربات های فروش VPN استفاده میکنند و کاربرای آنها پنل ایران است ، این موضوع است که هاست یا سرور خارج ربات تلگرامی با پنل ایران ارتباط برقرار نمی کند. این اسکریپت مشکل انها رفع خواهد کرد.  

این ابزار به‌صورت کاملاً خودکار اتصال پایدار بین ایران و خارج ایجاد می‌کند.

## 🧠 معماری عملکرد / ✨ امکانات

- ✅ نصب خودکار autossh
- ✅ ساخت خودکار SSH Key

- ✅ ایجاد سرویس systemd پایدار

- ✅ باز کردن خودکار پورت در فایروال

# 🚀 نصب سریع (پیشنهادی)

روی هر دو سرور (ایران و خارج) اجرا کنید:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/mobin-shahhoseini/leastBOT/main/install.sh)
```
run: ```leastbot```

---
📌 مراحل راه‌اندازی
🔹 مرحله 1 – تنظیم سرور خارج (KHAREJ)

روی سرور خارج اجرا کنید:

leastbot

گزینه زیر را انتخاب کنید:

2) Kharj (Server)

در این مرحله:

AllowTcpForwarding فعال می‌شود

GatewayPorts فعال می‌شود

پورت انتخابی در فایروال باز می‌شود

سرویس SSH ریستارت می‌شود

---
🔹 مرحله 2 – تنظیم سرور ایران (IRAN)

روی سرور ایران اجرا کنید:

leastbot

گزینه زیر را انتخاب کنید:

1) Iran (Client / Entry Point)

اطلاعات موردنیاز:

پورت پنل (مثلاً 2083 یا 80)

آی‌پی سرور خارج

یوزر SSH (پیش‌فرض root)

پورت SSH (پیش‌فرض 22)

اسکریپت به‌صورت خودکار:

SSH Key ایجاد می‌کند

کلید را به سرور خارج ارسال می‌کند

سرویس systemd ایجاد می‌کند

تونل را فعال می‌کند

🌐 نتیجه نهایی

پس از اتمام تنظیمات:

http://IP_KHARJ:PORT

پنل یا سرویس سرور ایران از طریق سرور خارج در دسترس خواهد بود 🔥

🛠 عیب‌یابی
❌ پورت روی خارج باز نمی‌شود
ss -lntp | grep PORT

اگر پورت توسط سرویس دیگری استفاده می‌شود، پورت متفاوت انتخاب کنید.

❌ اتصال SSH برقرار نمی‌شود

بررسی کنید:

پورت SSH باز باشد

یوزر صحیح وارد شده باشد

فایروال خارج SSH را بلاک نکرده باشد

❌ سرویس بالا نمی‌آید

journalctl -u leastbot-reverse-PORT -n 100
